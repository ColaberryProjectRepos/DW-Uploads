Deploying Project: (Step -3)

In this step we are going to mimic the actual deployment procedure that you would be doing in actual enterprise environment. The idea is to create your staging database, ODS and DW in Colaberry deployment server, in this step you will be creating those databases that you have already created in your “local” environment (personal computer).

Based on your project requirements go ahead and create those databases in the server you have already connected in previous step. Make sure you maintain the professionalism and give meaningful name of your databases, think about the scenarios where you will be giving access to people to use your data warehouse for analytical purpose.

Creating Databases and populate tables:

1.       Create all the required databases whichever applicable from below

a.       Staging

b.       ODS

c.       DW

2.       Create the required table in each database you have

3.       Populate tables in each database by using SSIS packages, you should already have the packages that populate tables in your local environment now you are going to change the connection to deployment server, adjust the folder paths etc. Run your packages from the Colaberry VM before moving to the next step. Colaberry Virtual machine has SSDT 2017 installed already, open your SSIS project and see you can run the packages. If they are running good and able to populate database tables then move on to the Deployment in Integration services catalogue.

 

Deploying SSIS packages:

As of now you should have created all the required databases and tables, now go to “Integration services catalog”, under SSISDB create a folder for your project (Give a meaningful name to identify your solution). Ex: YourFirstName_YourLastName_YourProjectName

  

 

Now Deploy your project in the designated folder you just created, you should already know how to deploy packages from the class.

Steps to Follow to deploy after creating the folder:

Right click on your project in visual studio and click on deploy (make sure the project is in ColaberryVM not in your local computer), it will open below screen.



Follow the prompt, path is going to point to your folder you have created.
