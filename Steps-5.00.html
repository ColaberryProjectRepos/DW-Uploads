Scheduling Packages (Step – 4)

The idea is to make sure you can run the packages and populate your data warehouse in Colaberry server without any dependencies on your local computer. That is why you have copied the ETL solution and data files in the Colaberry Virtual machine folders in previous steps.

In this step you are going to create SQL agent jobs the way you created in your local computer. The sql agent is already configured in the server, you will have to create your own job and steps for the packages you would run.

Once your deployment is completed, you should see the packages in the integration services catalog.

  

To execute these packages from SQL agent job you need to create a job and required steps, follow the naming conventions mentioned below. The job you are going to create needs to have recurring schedule that means you will also schedule the job to run certain time.

SQL agent job naming convention: YourFirstName_YourLastName_YourProjectName

 

 

SQL agent Job steps:

You will need to create the required job steps to populate your databases depending on the design of your packages.

 

 

Once all the required steps are created, now test your job and schedule to run the job daily, you can choose any time frame to execute your job.

***Do not schedule your job if it is not running successfully, you do not want to run a job daily which you know failing, that just doesn’t make any sense***

 

To schedule go to the job and click on “Schedules” and setup yours

 

That should complete your scheduling.
